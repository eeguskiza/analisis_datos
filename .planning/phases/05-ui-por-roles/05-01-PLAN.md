    2. El cuerpo de `can()` debe ser idéntico al patrón de 05-RESEARCH.md

       ```python
       def can(user: NexoUser | None, permission: str) -> bool:
           """True si ``user`` tiene ``permission``. Puro, sin side effects.

           Reglas (idénticas a ``require_permission`` — esta función ES la
           fuente de verdad; ``require_permission`` trampoliniza sobre ella):

           - ``user is None`` → False.
           - ``user.role == 'propietario'`` → True (bypass global).
           - Intersecta ``{d.code for d in user.departments}`` con
             ``PERMISSION_MAP[permission]``. True si hay intersección.

           Safe desde templates (Jinja global, D-03) y tests — sin async,
           sin HTTPException.
           """
           if user is None:
               return False
           if user.role == "propietario":
               return True
           allowed = PERMISSION_MAP.get(permission, [])
           user_depts = {d.code for d in user.departments}
           return bool(user_depts.intersection(allowed))
       ```

    3. REFACTORIZAR el cuerpo de `_check` (líneas 252-268) para trampolinar

       por:
       ```python
       if not can(user, permission):

               detail=f"Permiso requerido: {permission}",

       return user
       ```
