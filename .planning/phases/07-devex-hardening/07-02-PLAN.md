<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GH Actions runner -> repo | Runner efimero clona el repo, instala deps, ejecuta CI. Sin secretos persistentes. |
| Smoke test -> docker-compose | Smoke arranca compose con .env derivado de .env.example + NEXO_SECRET_KEY random. Sin credenciales reales. |
| Makefile targets -> host | `make test/lint/format` corren en el host del dev, no en el container. `make migrate` si usa compose exec. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-06 | Information Disclosure | coverage.xml subido como artifact puede revelar paths internos | mitigate | GH Actions artifacts son privados al repo. Paths son informacion estructural, no secretos. Retencion default (90d). Si el repo se hace publico en Mark-IV, reevaluar. |
| T-07-07 | Tampering | CI matrix con fail-fast: false permite que un job caido deje al otro completando su trabajo | accept | Intencional — queremos ver 3.11 y 3.12 independientes. Risk = gasto de minutos CI extra. |
| T-07-08 | Denial of Service | Smoke job podria colgar 45*2s = 90s esperando /api/health | mitigate | Loop tiene timeout duro via exit 1 tras 45 iteraciones. Step "Tear down" con `if: always()` garantiza limpieza aunque el run falle. |
| T-07-09 | Elevation of Privilege | `make migrate` ejecuta `docker compose exec web python ...` con lo que sea que este en `scripts/init_nexo_schema.py` | accept | El script es idempotente, con `ON CONFLICT DO NOTHING` y `CREATE SCHEMA IF NOT EXISTS`. Documentado en CLAUDE.md. No acepta input externo. |
| T-07-10 | Information Disclosure | `.env` sembrado en CI desde `.env.example` podria heredar valores placeholder con semblanza de credenciales | mitigate | `.env.example` (4500 bytes, verificado Phase 1) contiene solo placeholders documentados. El `NEXO_SECRET_KEY` de CI se genera aleatorio por run con openssl. No hay secretos reales persistidos. |
</threat_model>
