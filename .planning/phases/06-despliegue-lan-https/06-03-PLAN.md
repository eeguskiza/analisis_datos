## 1. Vista general

Nexo corre en un servidor Ubuntu Server 24.04 de la LAN ECS, accesible
desde cualquier equipo interno en `https://nexo.ecsmobility.local`. No
está expuesto a internet. Este runbook asume que el admin de respaldo
tiene conocimientos básicos de Linux pero NO del proyecto Nexo.

Tiempo de recuperación objetivo (RTO): 1-2 horas desde un Ubuntu limpio

- LAN interna ECS asumida de confianza moderada (no exposición internet).

## 8. Firewall (ufw) — reglas mínimas

[Copiar literal Topic 5 de RESEARCH.md líneas 657–690]

Secuencia segura (NO lockout):

```bash
sudo ufw allow from <SUBNET_LAN> to any port 22 proto tcp comment 'SSH from LAN'
sudo ufw allow 80/tcp  comment 'HTTP redirect to HTTPS'
sudo ufw allow 443/tcp comment 'HTTPS Nexo'
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw enable
sudo ufw status verbose
```

### 8.1 Advertencia crítica: Docker bypassa ufw

[Copiar párrafo literal Topic 5 §"Pitfall crítico" líneas 694–734]

Puntos clave a entender:
- `ufw deny 5432` NO cierra Postgres si `ports:` está publicado en el compose.
- En Nexo prod, Postgres queda cerrado porque `docker-compose.prod.yml` tiene
  `db.ports: !reset []` (Docker nunca publica el puerto al host).
- ufw protege SSH y cualquier servicio NO-Docker del host.

Verificación desde el propio servidor:

```bash
sudo ss -tlnp | grep :5432
# Esperado: sin output (puerto no escucha en el host)
```

Verificación desde un peer LAN:

```bash
nmap -Pn -p 22,80,443,5432 <IP_NEXO>
# Esperado: 22, 80, 443 open; 5432 closed o filtered
```

## 14. Landmines a no pisar

1. **NO usar `docker compose down -v`** — el flag `-v` borra el volumen `pgdata`
   y con él toda la BD de producción. Usar sólo `docker compose down` (o
   `make prod-down`). El target `make clean` del Makefile SÍ usa `-v` y está
   pensado SÓLO para dev; jamás ejecutarlo en prod.
2. **NO hacer `git commit` en el servidor de prod.** `scripts/deploy.sh` usa
   `git pull --ff-only`, que falla si hay commits locales. Cualquier cambio
   viene por merge a `main` en GitHub.
3. **NO copiar `caddy/Caddyfile` dev como `Caddyfile.prod`.** El dev tiene
   `auto_https disable_redirects` que rompe el redirect 80→443. El prod se
   escribe from-scratch (ver `caddy/Caddyfile.prod`).
4. **NO descomentar el bloque SMTP** en `.env` — SMTP está Out of Scope
   Mark-III (`# TODO Mark-IV`). Descomentarlo con valores falsos hace que
   pydantic-settings intente conectar a un servidor inexistente.
5. **NO confiar en que ufw cierra puertos de containers Docker.** Docker
   bypassa ufw. La defensa real del 5432 es `db.ports: !reset []` en el
   override prod (ver sección 8.1).
6. **NO exponer Nexo a internet.** LAN-only es decisión cerrada de Mark-III.
7. **NO rotar credenciales SQL Server durante un deploy ordinario** — se hace
   en una ventana de mantenimiento planificada.
8. **El root CA de Caddy expira en ~10 años.** Anotar la fecha (ver comando
   abajo). Cuando se acerque, Caddy emitirá una nueva y habrá que
   re-distribuir a todos los equipos.
   ```bash
   docker compose -f docker-compose.yml -f docker-compose.prod.yml exec caddy \
       openssl x509 -in /data/caddy/pki/authorities/local/root.crt -noout -enddate
   ```

```bash
#!/usr/bin/env bash
# tests/infra/deploy_smoke.sh — Checklist post-deploy Phase 6.
#
# Se ejecuta en el servidor Ubuntu tras `bash scripts/deploy.sh`.
# No corre en CI — requiere Docker + stack prod levantado + ufw configurado.
#
# Cada check imprime una linea:
#   [DEPLOY-XX] OK|FAIL: <mensaje>
#
# Exit 0 si todos los checks pasan. Exit N (count de fallos) en caso contrario.

set -uo pipefail
IFS=$'\n\t'

PROJECT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"
cd "${PROJECT_DIR}"

COMPOSE="docker compose -f docker-compose.yml -f docker-compose.prod.yml"
FAILS=0

check() {
    local req="$1"
    local msg="$2"
    local cmd="$3"
    if eval "${cmd}" >/dev/null 2>&1; then
        echo "[${req}] OK: ${msg}"
    else
        echo "[${req}] FAIL: ${msg} (cmd: ${cmd})"
        FAILS=$((FAILS + 1))
    fi
}

echo "=== Nexo deploy smoke — $(date -u +%FT%TZ) ==="

# DEPLOY-01: HTTPS via Caddy responde 200
check "DEPLOY-01" "HTTPS /api/health responde 200" \
    "curl -fs -m 10 -k https://nexo.ecsmobility.local/api/health"

# DEPLOY-01: cert emitido por Caddy local CA
check "DEPLOY-01" "cert emitido por Caddy Local Authority" \
    "openssl s_client -connect nexo.ecsmobility.local:443 -servername nexo.ecsmobility.local </dev/null 2>/dev/null | grep -q 'Caddy Local Authority'"

# DEPLOY-02: Postgres 5432 no escucha en el host
check "DEPLOY-02" "Postgres 5432 NO escucha en el host" \
    "! sudo ss -tlnp 2>/dev/null | grep -q ':5432 '"

# DEPLOY-02: psql via exec sigue funcionando
check "DEPLOY-02" "psql via docker compose exec OK" \
    "${COMPOSE} exec -T db bash -c 'psql -U \"\$POSTGRES_USER\" -d \"\$POSTGRES_DB\" -c \"SELECT 1;\"' | grep -q '1 row'"

# DEPLOY-03: web healthcheck healthy
check "DEPLOY-03" "web container healthcheck healthy" \
    "[ \"\$(docker inspect \$(${COMPOSE} ps -q web) --format '{{.State.Health.Status}}' 2>/dev/null)\" = 'healthy' ]"

# DEPLOY-03: caddy healthcheck healthy
check "DEPLOY-03" "caddy container healthcheck healthy" \
    "[ \"\$(docker inspect \$(${COMPOSE} ps -q caddy) --format '{{.State.Health.Status}}' 2>/dev/null)\" = 'healthy' ]"

# DEPLOY-03: restart policy unless-stopped
check "DEPLOY-03" "web restart policy = unless-stopped" \
    "[ \"\$(docker inspect \$(${COMPOSE} ps -q web) --format '{{.HostConfig.RestartPolicy.Name}}' 2>/dev/null)\" = 'unless-stopped' ]"

# DEPLOY-06: ufw activo
check "DEPLOY-06" "ufw activo" \
    "sudo ufw status 2>/dev/null | head -1 | grep -q 'Status: active'"

# DEPLOY-06: ufw allow 443
check "DEPLOY-06" "ufw permite 443/tcp" \
    "sudo ufw status 2>/dev/null | grep -qE '^443/tcp.*ALLOW'"

# DEPLOY-06: ufw allow 80
check "DEPLOY-06" "ufw permite 80/tcp (redirect)" \
    "sudo ufw status 2>/dev/null | grep -qE '^80/tcp.*ALLOW'"

# DEPLOY-06: ufw default deny incoming
check "DEPLOY-06" "ufw default incoming = deny" \
    "sudo ufw status verbose 2>/dev/null | grep -qE 'Default:.*deny \\(incoming\\)'"

echo "=== Fallos: ${FAILS} ==="
exit "${FAILS}"
```
